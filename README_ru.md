# Cursor Memory Bank v1.1

> **Система управления контекстом с поддержкой ИИ и командой ПРОДОЛЖАЙ**

Комплексная система управления памятью и контекстом, которая трансформирует работу ИИ-ассистентов в Cursor IDE. Memory Bank предоставляет структурированные рабочие процессы, интеллектуальное управление задачами, постоянный контекст между сессиями разработки и бесшовное возобновление работы с революционной командой ПРОДОЛЖАЙ.

## 🚀 Быстрый старт

Запустите систему менее чем за 5 минут:

### Предварительные требования

- Установленная [Cursor IDE](https://cursor.sh/)
- Базовое знакомство с разработкой с помощью ИИ

### Установка

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/your-username/cursor-memory-bank.git
   cd cursor-memory-bank
   ```

2. **Настройте Memory Bank**
   ```bash
   # Скопируйте правила в конфигурацию Cursor
   cp -r rules/ .cursor/rules/

   # Инициализируйте структуру Memory Bank
   mkdir -p memory-bank/{archive,creative,development,reflection}
   ```

3. **Проверьте установку**
   ```bash
   # Проверьте, что правила установлены корректно
   ls .cursor/rules/isolation_rules/
   ```

4. **Начните использовать Memory Bank**
   - Откройте Ваш проект в Cursor IDE
   - Введите `VAN` для инициализации Memory Bank для любой задачи
   - Следуйте управляемому рабочему процессу

## 🔧 Как это работает

Memory Bank работает через сложную систему на основе режимов с автоматическим управлением контекстом:

```mermaid
graph TD
    Start([🚀 Начать задачу]) --> VAN[🔍 VAN<br>Инициализация и анализ]
    VAN --> SaveContext{💾 Сохранить контекст}
    SaveContext --> PLAN[📋 PLAN<br>Стратегия и архитектура]
    PLAN --> CREATIVE[🎨 CREATIVE<br>Дизайн и решения]
    CREATIVE --> IMPLEMENT[🔨 IMPLEMENT<br>Сборка и код]
    IMPLEMENT --> QA[✅ QA<br>Тестирование и проверка]
    QA --> REFLECT[📚 REFLECT<br>Обучение и документация]
    REFLECT --> Complete([✅ Задача завершена])

    %% Управление контекстом
    SaveContext -.-> ContextFile[(📄 current-context.md)]
    ContextFile -.-> Continue[🔄 Команда ПРОДОЛЖАЙ]
    Continue -.-> RestoreMode{🎯 Восстановить режим}
    RestoreMode --> PLAN
    RestoreMode --> CREATIVE
    RestoreMode --> IMPLEMENT
    RestoreMode --> QA
    RestoreMode --> REFLECT

    %% Универсальный режим
    VAN --> UNIVERSAL[🌐 UNIVERSAL<br>Авто-рабочий процесс]
    UNIVERSAL --> AutoTransition{🔄 Авто переходы<br>между режимами}
    AutoTransition --> PLAN
    AutoTransition --> CREATIVE
    AutoTransition --> IMPLEMENT
    AutoTransition --> QA
    AutoTransition --> REFLECT

    %% Обработка прерываний
    PLAN -.-> Interrupt([⚡ Прерывание])
    CREATIVE -.-> Interrupt
    IMPLEMENT -.-> Interrupt
    QA -.-> Interrupt
    REFLECT -.-> Interrupt
    Interrupt -.-> SaveState[💾 Сохранить текущее состояние]
    SaveState -.-> ContextFile

    style SaveContext fill:#e1f5fe
    style ContextFile fill:#f3e5f5
    style Continue fill:#e8f5e8
    style UNIVERSAL fill:#fff3e0
```

### Обзор режимов

- **🔍 VAN Mode**: Инициализация задач, оценка сложности и обнаружение проекта
- **📋 PLAN Mode**: Стратегическое планирование и проектирование архитектуры
- **🎨 CREATIVE Mode**: Творческое решение проблем и дизайнерские решения
- **🔨 IMPLEMENT Mode**: Реализация кода и разработка
- **✅ QA Mode**: Обеспечение качества и тестирование
- **📚 REFLECT Mode**: Обучение и захват знаний
- **🌐 UNIVERSAL Mode**: Автоматическое управление рабочими процессами с интеллектуальными переходами

### Возможности управления контекстом

- **💾 Автоматическое сохранение контекста**: Каждый переход режима сохраняет текущее состояние
- **🔄 Команда ПРОДОЛЖАЙ**: Возобновление работы с любой точки прерывания
- **📄 Постоянное состояние**: Контекст поддерживается в `memory-bank/system/current-context.md`
- **🎯 Умное восстановление**: Автоматически возвращается к правильному режиму и фазе
- **📊 Отслеживание прогресса**: Мониторинг статуса завершения задач в реальном времени

## 🎯 Новые команды

### Команды управления контекстом
- **`ПРОДОЛЖАЙ`** / **`CONTINUE`** - Возобновить прерванную работу
- **`ПОКАЗАТЬ КОНТЕКСТ`** - Отобразить текущий контекст и прогресс
- **`ОЧИСТИТЬ КОНТЕКСТ`** - Очистить сохраненный контекст (начать заново)
- **`СКАНИРОВАТЬ ПРАВИЛА`** - Обнаружить и интегрировать правила проекта
- **`НАЙТИ ЗАДАЧИ`** - Поиск элементов TODO/FIXME в документации

### Возможности обнаружения проекта
Memory Bank теперь автоматически обнаруживает:
- **Правила проекта**: Сканирует файлы `rules/*.md` для руководящих принципов проекта
- **TODO элементы**: Находит маркеры `TODO`, `FIXME`, `HACK` в документации
- **Незавершенные задачи**: Обнаруживает непроверенные элементы `- [ ]`
- **Русские задачи**: Распознает маркеры `нужно`, `требуется`, `добавить`, `исправить`
- **Контекст файлов**: Включает размеры файлов и количество строк для лучшего понимания
